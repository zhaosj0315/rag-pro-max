# 自动测试配置

## ✅ 已配置

Git Hook 已自动安装！每次 `git commit` 都会自动运行出厂测试。

## 🚀 工作流程

```bash
# 1. 修改代码
vim src/apppro.py

# 2. 添加到暂存区
git add src/apppro.py

# 3. 提交（自动运行测试）
git commit -m "修复某个功能"

# 输出示例：
# 🔍 运行出厂测试...
# ============================================================
#   RAG Pro Max 出厂测试
# ============================================================
# ✅ 通过: 64/67
# ✅ 出厂测试通过！继续提交...
```

## 🔧 特殊情况

### 跳过测试（紧急情况）

```bash
git commit --no-verify -m "紧急修复"
```

### 手动运行测试

```bash
./scripts/test.sh
# 或
python3 tests/factory_test.py
```

## 📋 测试失败时

如果测试失败，提交会被阻止：

```
❌ 出厂测试失败！
💡 提示：修复问题后再提交，或使用 git commit --no-verify 跳过测试
```

**处理步骤：**

1. 查看失败的测试项
2. 修复问题
3. 重新提交
