## 🛠️ 企业集成指南

### 🔌 API集成
#### RESTful API
```bash
# 启动API服务
python src/api/fastapi_server.py

# 主要端点
GET  /health              # 健康检查
POST /query               # 查询接口
GET  /knowledge-bases     # 知识库列表
POST /upload              # 文件上传
```

#### 核心接口示例
```python
# 查询接口
curl -X POST "http://localhost:8000/query" \
  -H "Content-Type: application/json" \
  -d '{"query": "企业部署要求", "kb_name": "企业文档"}'

# 文件上传
curl -X POST "http://localhost:8000/upload" \
  -F "file=@document.pdf" \
  -F "kb_name=企业文档"
```

## ❓ 企业常见问题

### 🏢 部署相关
**Q: 是否支持完全离线部署？**
A: 是的，RAG Pro Max支持完全离线部署，使用本地Ollama模型，数据不出内网。

**Q: 企业级部署的硬件要求？**
A: 推荐配置：16GB+ RAM, 50GB+ SSD, 支持GPU加速，具体需求根据知识库规模调整。

**Q: 是否支持集群部署？**
A: 支持Docker Swarm和Kubernetes集群部署，可实现高可用和负载均衡。

### 🔧 功能使用
**Q: 支持哪些文档格式？**
A: 支持15+种格式：PDF, Word, Excel, PPT, TXT, MD, HTML等，包括扫描版PDF的OCR识别。

**Q: 如何提高查询准确性？**
A: 建议开启"深度思考"和"智能研究"模式，使用具体明确的问题描述。

**Q: 是否支持批量文档处理？**
A: 支持文件夹批量上传，自动OCR识别，元数据提取和标签分类。

### 🛡️ 安全合规
**Q: 数据安全如何保障？**
A: 所有数据本地存储，支持加密，完全离线运行，符合企业安全要求。

**Q: 是否符合行业合规要求？**
A: 支持GDPR、等保三级等合规要求，提供完整的审计日志。

**Q: 如何进行数据备份？**
A: 支持自动备份机制，可定期备份知识库和配置信息。
